## 예측에서 분석으로: 브랜딩의 전환점

스포츠 데이터를 활용한 서비스를 운영하다 보면 언젠가는 마주하게 되는 중요한 순간이 있습니다. **법적 리스크와 사회적 책임**에 대한 고민이죠. KBO Predictor를 개발하면서 머신러닝과 AI 에이전트의 기술적 구현에만 집중하다가, 서비스가 가질 수 있는 잠재적 위험성을 깊이 있게 검토하게 되었습니다.

가장 큰 변화는 서비스 정체성의 전환이었습니다. "KBO Predictor"에서 **"KBO AI Analyzer"**로, "예측"에서 "분석"으로 용어를 전면 변경했습니다. 단순한 단어 교체가 아니라 서비스 철학의 근본적 변화였습니다.

```tsx
// Before: 예측 중심의 UI
<button>Predict</button>
<div>AI 예측 승리팀</div>
<span>예측 실패</span>

// After: 분석 중심의 UI
<button>Analyze</button>
<div>AI 분석 결과</div>
<span>분석 실패</span>
```

## 포괄적인 법적 페이지 구축

기술 중심의 개발자에게 법무 문서 작성은 새로운 도전이었습니다. 하지만 사용자와 서비스를 보호하기 위해 세 가지 핵심 페이지를 구축했습니다.

**면책 조항**에서는 서비스의 본질을 명확히 정의했습니다. "통계 데이터 및 인공지능 기반의 스포츠 분석 정보"라고 명시하며, 도박 목적 사용을 엄격히 금지했습니다. 특히 AI 에이전트 토론 결과가 언어 모델의 생성물이며 사실과 다를 수 있다는 점까지 투명하게 공개했습니다.

**이용약관**에서는 서비스를 "도박, 베팅 또는 사행 행위를 위한 서비스가 아닌 스포츠 분석 플랫폼"으로 정의했습니다. 기술적 구현체에 대한 지적재산권도 명확히 했습니다 - AI 에이전트 토론 결과와 분석 모델은 운영자 소유이지만, 경기 결과나 선수 성적 같은 사실 데이터는 공공 정보로 구분했습니다.

```tsx
export default function DisclaimerPage() {
  return (
    <div className="max-w-3xl mx-auto px-6 py-10">
      <section>
        <h2>3. 금지 사항</h2>
        <div className="bg-red-950/20 border border-red-900/30 rounded-lg p-4">
          <p className="text-red-300 font-semibold">본 서비스의 분석 정보를 다음 목적으로 사용하는 것을 엄격히 금지합니다:</p>
          <ul>
            <li>불법 도박 또는 사행 행위</li>
            <li>불법 스포츠 베팅</li>
          </ul>
        </div>
      </section>
    </div>
  );
}
```

## 사회적 책임: 도박 문제 예방

가장 중요하게 생각한 부분은 도박 중독 예방이었습니다. 단순히 금지 조항을 명시하는 것을 넘어, 실질적인 도움을 제공하고자 했습니다. **한국도박문제관리센터 1336번**을 여러 곳에 노출시키고, 푸터에 상시 노출되는 면책 배너를 추가했습니다.

이 배너는 모든 페이지 하단에 표시되어 사용자들이 서비스의 성격을 지속적으로 인지할 수 있도록 했습니다. 기술적으로는 간단한 구현이지만, 사용자 보호 측면에서는 매우 중요한 기능입니다.

## 개인정보처리방침의 투명성

현재 회원가입이 없는 서비스 특성을 살려 **최소한의 데이터만 수집**한다는 점을 명확히 했습니다. IP 주소와 접속 로그만 3개월 보관하며, 외부 API 사용 시에도 개인정보가 아닌 경기 데이터만 전송한다고 명시했습니다.

OpenAI와 Anthropic API 사용에 대해서도 투명하게 공개했습니다. AI 분석 기능이 핵심인 서비스에서 사용자들이 자신의 데이터가 어떻게 처리되는지 알 권리가 있다고 생각했습니다.

## 기술과 법무의 균형점

이번 작업을 통해 깨달은 것은 좋은 기술도 올바른 사용 맥락 안에서 가치를 발휘한다는 점입니다. 머신러닝 모델의 정확도를 높이는 것만큼이나, 그 결과물이 사회에 미칠 영향을 고려하는 것도 개발자의 중요한 책임입니다.

법적 컴플라이언스 구축은 단순한 리스크 관리를 넘어 **서비스 가치의 재정립** 과정이었습니다. 예측에서 분석으로의 전환은 기술의 본질을 해치지 않으면서도 더 건전한 서비스로 거듭날 수 있는 길을 열어주었습니다.
