## 프로젝트 소개

최근 완성한 '배당1초' 프로젝트는 부동산 전문가들의 오랜 고민을 해결하는 서비스입니다. 기존에 전문가가 엑셀로 30분간 수동 입력하며 분석하던 NPL(부실채권) 배당 계산을 **단 1초 만에** 자동화했습니다.

등기부등본 PDF를 업로드하면 갑구/을구 권리 정보를 파싱하고, 법정 배당 순위에 따라 정확한 배당금과 수익률을 계산해 리포트로 제공하는 B2B/B2H 웹 서비스입니다.

## 핵심 아키텍처 설계

### 마이크로서비스 구조

복잡한 배당 계산 로직을 안정적으로 처리하기 위해 다음과 같은 3-tier 구조를 선택했습니다:

- **Frontend (Next.js 14)**: PDF 업로드, 실시간 상태 추적, 결제, 리포트 대시보드
- **Backend Core (Spring Boot 3.2)**: 배당 계산 엔진, 결제 처리, 비즈니스 로직
- **Backend Parsing (FastAPI)**: PDF 파싱, OCR, 권리 정보 추출

각 서비스가 독립적으로 배포되어 확장성과 장애 격리를 보장합니다.

### 실시간 상태 추적 시스템

PDF 파싱부터 배당 계산까지의 전 과정을 사용자가 실시간으로 모니터링할 수 있도록 **Server-Sent Events(SSE)**를 구현했습니다:

```java
@GetMapping(value = "/{analysisId}/status/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter streamStatus(@PathVariable UUID analysisId) {
    SseEmitter emitter = new SseEmitter(120_000L);
    
    sseScheduler.scheduleAtFixedRate(() -> {
        String statusJson = redisStatusService.getStatus(analysisId);
        if (statusJson != null) {
            emitter.send(SseEmitter.event().name("status").data(statusJson));
        }
    }, 0, 1, TimeUnit.SECONDS);
    
    return emitter;
}
```

Redis를 상태 저장소로 활용해 각 단계별 진행 상황을 실시간으로 전송합니다.

## 배당 계산 엔진의 핵심

### 법정 배당 순위 구현

가장 복잡한 부분은 민사집행법에 따른 정확한 배당 순위 계산입니다. 다음과 같은 5단계 순위를 BigDecimal로 정밀 계산합니다:

```java
public class DividendRuleEngine {
    
    public DividendCalculationResult calculate(CalculationParams params) {
        BigDecimal remainingAmount = params.getExpectedWinningBid();
        List<DistributionItemResult> results = new ArrayList<>();
        
        // 1순위: 경매집행비용 (낙찰가의 3%)
        remainingAmount = distributeRank1(remainingAmount, results, params);
        
        // 2순위: 당해세 (국세 + 지방세)
        remainingAmount = distributeRank2(remainingAmount, results, params);
        
        // 3순위: 최우선변제금 (소액임차인)
        remainingAmount = distributeRank3(remainingAmount, results, params);
        
        // 4순위: 담보물권 (근저당, 전세권 등)
        remainingAmount = distributeRank4(remainingAmount, results, params);
        
        // 5순위: 일반채권 (안분배당)
        distributeRank5(remainingAmount, results, params);
        
        return new DividendCalculationResult(results, calculateTotalDistribution(results));
    }
}
```

## PDF 파싱의 도전과 해결

### 권리 정보 정규화

등기부등본의 권리 유형은 표기가 다양하고 불규칙합니다. 이를 표준화하는 정규화 로직을 구현했습니다:

```python
def normalize_right_type(raw_type: str) -> RightType:
    """원본 권리명을 표준 RightType으로 정규화"""
    
    # 근저당권 패턴
    if re.search(r'근저당', raw_type, re.IGNORECASE):
        return RightType.MORTGAGE
    
    # 전세권 패턴    
    if re.search(r'전세', raw_type, re.IGNORECASE):
        return RightType.LEASE
    
    # 가압류 패턴
    if re.search(r'가압류', raw_type, re.IGNORECASE):
        return RightType.PROVISIONAL_SEIZURE
        
    return RightType.GENERAL_CLAIM
```

## 사용자 경험 최적화

### 비동기 처리와 상태 관리

PDF 파싱과 배당 계산이 복합적으로 진행되는 과정을 사용자가 명확히 이해할 수 있도록 단계별 상태를 제공합니다:

```typescript
export const useAnalysisStatus = (analysisId: string) => {
  const [status, setStatus] = useState<AnalysisStatus>(AnalysisStatus.PENDING)
  const [progress, setProgress] = useState(0)
  const [error, setError] = useState<string | null>(null)

  useEffect(() => {
    const eventSource = new EventSource(`/api/analysis/${analysisId}/status/stream`)
    
    eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data)
      setStatus(data.status)
      setProgress(data.progress)
    }

    return () => eventSource.close()
  }, [analysisId])

  return { status, progress, error }
}
```

계산이 완료된 후 결제를 통해서만 리포트에 접근할 수 있도록 Toss Payments를 연동했습니다:

```java
@Service
public class TossPaymentClient {
    
    public PaymentResult confirmPayment(String paymentKey, String orderId, Long amount) {
        HttpHeaders headers = new HttpHeaders()
        headers.setBasicAuth(tossSecretKey + ":", "")
        
        PaymentConfirmRequest request = new PaymentConfirmRequest(orderId, amount)
        
        return restTemplate.postForObject(
            tossApiUrl + "/payments/confirm/" + paymentKey,
            new HttpEntity<>(request, headers),
            PaymentResult.class
        )
    }
}
```
