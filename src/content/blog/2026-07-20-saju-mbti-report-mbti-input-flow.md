## 왜 바꿨나

서비스 초기 설계는 모든 사용자에게 10문항짜리 상황극 퀴즈를 완주하게 만들었다. 사용자가 직접 MBTI를 알고 있더라도 예외가 없었다. 데이터를 보니 실상은 달랐다. MBTI를 이미 아는 사람이 압도적 다수였고, 10문항은 그들에게 그냥 장벽이었다.

이번 Phase 4A·4B의 핵심 결정은 단순하다. **주경로와 폴백을 바꾼다.** 16개 코드를 바로 고르는 그리드가 첫 화면이 되고, 모르는 사람만 '모르겠어요 · 성향 찾기' 버튼으로 퀴즈에 진입한다.

## 상태 설계: Source 타입 하나로 분기

구조 변경 자체는 크지 않았다. 가장 핵심적인 추가는 `Source` 타입이다.

```typescript
type Phase = 'idle' | 'quiz' | 'done';
/** 값의 출처 — 완료 문구를 가른다(직접 고름 vs 퀴즈로 찾음). */
type Source = 'direct' | 'quiz';
```

`source` 상태 하나가 완료 화면의 분기 전체를 제어한다. 직접 고른 경우에는 코드만 보여주고, 퀴즈로 찾은 경우에만 "반복해서 마음이 향한 쪽을 읽었어요"라는 해석 문구를 덧붙인다. 작은 차이지만 사용자 입장에서는 자기가 고른 것과 진단된 것을 구분해 받아들일 수 있게 된다.

또 하나 챙긴 것은 `restart` 동작이다. 이전에는 다시 하기를 누르면 곧바로 퀴즈(`phase: 'quiz'`)로 돌아갔다. 이제는 `idle`로 복귀해 그리드 선택과 퀴즈 중 다시 고를 수 있다.

```typescript
function restart() {
  setAnswers([]);
  setMbti('');
  setPhase('idle'); // 이전: 'quiz'
}
```

## 퀴즈 문항 전면 재작성: 세계관 → 실생활

기존 문항은 "알 껍데기에 작은 금이 갑니다", "마을 어른이 별우물 이야기를 들려줍니다" 같은 판타지 세계관 기반이었다. 리브랜드 v2 방향은 실제 생활 장면으로의 전환이다.

10개 장면을 모두 바꿨다. 새 모임 첫날, 퇴근 후 저녁, 탕비실, 설명서 없는 물건, 길 묻는 사람, 드라마 마지막 화, 여행 짐 꾸리기. 누구나 한 번쯤 겪어 본 상황들이다.

중요한 제약이 있었다. **축 매핑(poles)은 한 글자도 건드리지 않는다.** 각 선택지가 어떤 MBTI 축을 대변하는지는 그대로 보존해야 `scoreMbti`의 진단 결과가 바뀌지 않는다. 문항 텍스트는 전부 갈아엎으면서 내부 로직은 완전히 동일하게 유지하는 작업이었다.

테스트가 이 불변식을 명시적으로 잡는다.

```typescript
it('문항 안에서는 검사 체계를 드러내지 않고 실생활 장면으로만 묻는다', () => {
  const copy = QUESTIONS.flatMap((q) => [q.scene, q.prompt, ...q.options.map((o) => o.label)]).join(' ');
  expect(copy).not.toMatch(/MBTI|성격유형|4글자|코드/i);
  // 판타지 세계관 어휘를 쓰지 않는다(리브랜드 v2 — 실제 생활 장면 기반).
  expect(copy).not.toMatch(/마을|둥지|껍데기|도깨비|우물|사주몬/);
});
```

## 코드 비노출 정책 반전

이전에는 내부적으로 MBTI 코드를 도출해도 사용자 화면에는 노출하지 않는 정책이었다. 완료 문구도 "지금의 성향을 찾았어요. 타고난 기운과 함께 사주몬에 담을게요." 처럼 코드를 숨겼다.

이번에 정책이 반전됐다. 완료 화면에서 **INFJ**, **ENFP** 처럼 코드를 그대로 보여준다. 사용자가 이미 아는 개념을 굳이 가리는 것이 오히려 어색하다는 판단이다.

## 검증

전체 483개 테스트가 통과한다. 신규 추가된 5개는 직접 선택, 폴백 진입, idle 복귀, 퀴즈 완주, 판타지 어휘 부재를 각각 커버한다. `next build`와 tsc, eslint 모두 클린하다.

구조 변경이 작았던 이유는 처음부터 `phase` 상태로 분기를 관리하고 있었기 때문이다. 새로운 경로를 추가할 때 기존 퀴즈 로직을 손댈 필요가 없었다. 설계가 확장에 열려 있었다는 것을 확인한 작업이기도 했다.
