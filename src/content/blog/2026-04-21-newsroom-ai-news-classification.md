## AI가 만든 분류와 현실의 괴리

**Newsroom AI** 프로젝트를 운영하면서 흥미로운 문제에 부딪혔다. 시스템에 통합된 Claude Haiku가 의료·보건 관련 기사를 'health'라는 카테고리로 분류하면, 백엔드에서 422 오류가 발생하며 요청이 실패했다. 언뜻 보면 합리적인 분류 같은데 왜 오류가 날까?

문제의 핵심은 **사전 정의된 카테고리 체계**와 AI의 자유로운 분류 결과 간의 불일치였다. 한국의 언론사들은 전통적으로 의료·보건 기사를 '사회' 섹션에 배치하는 편집 관행을 가지고 있다. 하지만 Claude Haiku는 이런 한국적 맥락을 모른 채 서구적 기준으로 'health'나 'medical' 카테고리를 생성했던 것이다.

## 카테고리 정규화로 문화적 차이 해결

이 문제를 해결하기 위해 **카테고리 매핑 시스템**을 확장했다. AI가 생성한 원시 카테고리를 한국 언론 환경에 맞게 변환하는 정규화 로직이다.

```python
CATEGORY_ALIASES = {
    "business": "economy",
    "finance": "economy",
    "social": "society",
    "health": "society",  # 의료·보건 기사 — 한국 편집 관행상 society 로 분류
    "medical": "society",
    "world news": "world",
    "international": "world",
    "cultural": "culture",
}
```

이제 AI가 'health'나 'medical'로 분류한 기사들은 자동으로 'society' 카테고리로 매핑된다. 단순해 보이지만, 이런 **문화적 지역화**야말로 실제 서비스에서 가장 중요한 부분이다.

## 사용자 경험을 위한 정렬 옵션 확장

또 다른 개선사항은 뉴스 목록 API의 정렬 기능이었다. 기존에는 중요도, 발행일, 생성일로만 정렬할 수 있었는데, **수집일(collected_at)** 기준 정렬이 추가됐다.

```python
sort_by: str = Query(
    "importance",
    pattern="^(importance|published_at|created_at|collected_at)$",
)
```

이 변경의 배경에는 실사용자들의 니즈가 있었다. 뉴스의 '발행일'과 '수집일'은 다를 수 있고, 특히 해외 뉴스나 지연 배포되는 기사의 경우 시스템에 들어온 순서로 보고 싶어하는 사용자들이 있었기 때문이다.

## 작은 버그픽스가 주는 인사이트

이번 커밋은 겉보기에는 간단한 버그 수정이지만, 실제로는 **AI 시스템의 현실 적용**에서 마주하는 복잡함을 보여준다. 기술적으로 완벽한 AI 분류 결과라도 실제 사용 환경의 문화적, 제도적 맥락과 맞지 않으면 오히려 장애물이 된다.

특히 한국어 뉴스 데이터를 다루면서 느끼는 점은, 서구 중심적으로 학습된 AI 모델들이 한국의 언론 관행을 제대로 반영하지 못한다는 것이다. 이런 간극을 메우는 것이 바로 엔지니어의 역할이고, 단순한 매핑 로직 하나도 깊은 도메인 이해가 필요한 작업이었다.

결국 좋은 AI 서비스는 **기술의 완성도**만큼이나 그것을 현실에 맞게 조정하는 **세심한 엔지니어링**이 중요하다는 걸 다시 한번 확인했다.
