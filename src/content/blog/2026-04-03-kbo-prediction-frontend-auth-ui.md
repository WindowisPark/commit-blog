## 왜 구독 시스템이 필요했는가

KBO AI 예측 서비스를 운영하면서 두 가지 문제에 부딪혔습니다. 첫째, 모든 고급 분석 결과를 무료로 제공하면 서비스 지속 가능성에 한계가 있었고, 둘째, 사용자들에게 단계적으로 가치를 전달할 방법이 필요했습니다.

그래서 **Free, Basic, Pro** 세 단계의 구독 시스템을 설계했습니다. Free는 매일 1회 기본 예측, Basic은 승률과 분석 근거 미리보기, Pro는 ML 모델 상세와 AI 토론 전문까지 제공하는 구조로요.

## 사용자 인증 시스템 구축

먼저 전체 앱을 감싸는 **AuthProvider**를 구현했습니다. React Context를 활용해 로그인 상태와 사용자 정보를 전역에서 관리하고, 토큰 자동 갱신까지 처리하도록 설계했습니다.

```tsx
const { user, loading, logout } = useAuth();

// 토큰이 있으면 API 요청 헤더에 자동 추가
if (token) authHeaders["Authorization"] = `Bearer ${token}`;
```

로그인 페이지는 하나의 컴포넌트에서 로그인과 회원가입을 탭으로 전환할 수 있게 만들었습니다. 사용자 경험을 고려해 에러 처리와 로딩 상태까지 세심하게 구현했고, 가입 시 Free 플랜이 자동 적용된다는 안내도 추가했습니다.

## 티어 기반 콘텐츠 제어 시스템

가장 까다로웠던 부분은 **TierGate** 컴포넌트 구현이었습니다. 사용자의 현재 티어와 콘텐츠가 요구하는 최소 티어를 비교해서, 권한이 없으면 블러 처리된 플레이스홀더를 보여주는 방식으로 설계했습니다.

```tsx
<TierGate tier={userTier} requiredTier="pro" placeholder={
  <ModelBar prob={0.55} label="XGBoost" blurred={true} />
}>
  {/* Pro 사용자만 볼 수 있는 실제 ML 모델 결과 */}
  <ModelBar prob={realData.xgboost} label="XGBoost" />
</TierGate>
```

**BlurredValue** 컴포넌트는 숫자나 텍스트를 자연스럽게 가려주는 역할을 합니다. 승률이나 분석 근거처럼 핵심 정보를 "맛보기"로 제공하되, 전체 내용은 유료 플랜에서만 볼 수 있게 했습니다.

## 예측 결과 표시 방식 개선

기존에는 홈팀과 어웨이팀의 승률을 따로 표시했는데, 이게 사용자에게 혼란을 줄 수 있다는 걸 깨달았습니다. 대신 **예상 승리팀의 우세 확률**만 표시하는 방식으로 변경했습니다.

```tsx
// 우세 확률: predicted_winner 기준으로 항상 50%+
const winnerProb = homeProb != null ? (isHomeWin ? homeProb : 1 - homeProb) : null;
```

"KIA 60% 우세"처럼 직관적으로 보여주되, Free 사용자에게는 확률 부분을 블러 처리해서 업그레이드 동기를 부여했습니다.

## 사용자 온보딩과 전환 최적화

비로그인 사용자를 위해 메인 페이지에 **히어로 배너**를 추가했습니다. 서비스 소개와 함께 "무료로 시작하기" 버튼을 눈에 띄게 배치하고, "매일 무료 1회 분석", "가입 10초", "카드 등록 불필요" 같은 진입 장벽을 낮추는 메시지를 강조했습니다.

로그인한 사용자에게는 기존의 깔끔한 헤더를 그대로 유지해서 서비스 이용에 집중할 수 있게 했습니다.

## 마이페이지와 플랜 비교

마이페이지에서는 현재 플랜 상태를 한눈에 보여주고, 세 가지 플랜을 카드 형태로 비교할 수 있게 구성했습니다. 각 플랜의 기능을 체크마크와 X 표시로 구분해서 표시하고, 현재 이용 중인 플랜은 특별한 강조 효과를 적용했습니다.

업그레이드 버튼을 눌러도 아직은 "결제 시스템 준비 중"이라는 알림이 뜨지만, UI 구조는 실제 결제 연동을 염두에 두고 확장 가능하게 설계했습니다.

## 기술적 도전과 해결

가장 까다로웠던 건 API 요청에 토큰을 자동으로 붙이는 부분이었습니다. 처음에는 예측 요청에서 인증 헤더가 누락되어 Free 사용자도 전체 데이터를 받을 수 있는 버그가 있었는데, fetch 함수에서 토큰 존재 여부를 확인해서 헤더를 동적으로 구성하는 방식으로 해결했습니다.

또한 예측 카드 미리보기에서도 홈팀 확률이 아닌 승리팀 확률을 표시하도록 로직을 수정해서 일관성 있는 사용자 경험을 만들었습니다.

이런 구독 시스템을 통해 사용자는 부담 없이 서비스를 체험하고, 가치를 느끼면 자연스럽게 업그레이드할 수 있는 구조가 완성되었습니다.
