## 문제 인식: 정체성 없는 대시보드

**Newsroom AI**는 처음에 전형적인 SaaS 스타일로 개발되었습니다. Geist 폰트, 무채색 팔레트, 둥근 모서리의 카드들이 어떤 회사든 쓸 법한 범용적인 느낌을 주었죠. 그런데 실제 서울신문 편집진이 사용한다면? 브랜드 정체성이 전혀 드러나지 않는 문제가 있었습니다.

첫 번째 큰 변화는 **디자인 시스템의 전면 개편**이었습니다. Pretendard 폰트로 교체하고, 서울신문의 브랜드 컬러인 파란색(`#0066CC`)을 primary 색상으로 도입했습니다. 또한 신문의 직선적 기하학에 맞춰 border-radius를 0.625rem에서 0.25rem로 줄였습니다.

```css
:root {
  --font-sans: Pretendard, var(--font-geist-sans), -apple-system;
  --primary: oklch(0.48 0.17 253); /* 서울신문 파랑 */
  --radius: 0.25rem; /* 신문 직선 기하 */
}
```

## 신문 편집 관례를 UI에 적용

단순히 색상만 바꾸는 게 아니라, 실제 **신문 제작 관례**를 디지털 인터페이스에 녹여냈습니다. 사이드바의 "Newsroom AI"를 "서울신문 / AI 뉴스룸" 2단 마스트헤드로 변경하고, 섹션 헤더에는 신문에서 사용하는 uppercase tracking-wider 스타일을 적용했습니다.

가장 눈에 띄는 변화는 **뉴스 페이지의 구조화**였습니다. 기존의 평면적인 리스트 대신 신문 1면 규약을 적용해 Top 3 히어로 카드와 카테고리별 섹션 구분을 도입했습니다. 이렇게 하니 사용자가 한 번에 전체 뉴스 구조를 파악할 수 있게 되었죠.

```tsx
// Top 3 히어로 + 나머지를 카테고리 그룹핑
const { hero, grouped } = useMemo(() => {
  const hero = articles.slice(0, 3);
  const rest = articles.slice(3);
  const byCat: Record<string, Article[]> = {};
  for (const a of rest) {
    const k = a.analysis?.category ?? "unknown";
    (byCat[k] ??= []).push(a);
  }
  // 정치/경제/사회/국제/기술/문화/스포츠 순서로 정렬
}, [articles]);
```

## 실용성 중심의 UX 개선

디자인만큼이나 중요한 건 **실제 기자들의 워크플로우** 개선이었습니다. 기존에는 뉴스 목록에서 각 기사를 클릭해 상세 페이지로 들어가야만 초안 생성이 가능했는데, 이를 카드 레벨에서 바로 접근할 수 있도록 **DraftDialog 버튼을 직접 노출**했습니다.

또한 절대시간 표기("2026-04-20 오후 8:13")를 "14분 전" 같은 **상대시간으로 변경**해 정보 스캔 속도를 높였습니다. 오늘 수집된 전체 기사 수와 중요 기사 수를 보여주는 요약 스트립도 추가해, 한눈에 뉴스룸 상황을 파악할 수 있게 했죠.

## 정보 노이즈 제거하기

개발 과정에서 발견한 흥미로운 점은 **"더 많은 정보가 항상 좋은 건 아니다"**였습니다. 초기 버전에서는 감성 분석(긍정/부정/중립) 배지나 정확한 중요도 점수(8.7/10) 같은 세부 정보들을 UI에 노출했는데, 실제로는 편향 프레이밍 오해를 유발하거나 단일 LLM의 주관적 판단에 과도하게 의존하게 만드는 부작용이 있었습니다.

대신 집합 신호("매체 N곳, 기사 M건") 같은 **객관적 지표**로 교체하고, "속보"라는 과장된 표현도 "주요 기사 감지"로 순화했습니다. 색상도 경고성이 강한 빨간색에서 좀 더 차분한 amber 톤으로 조정했고요.

## 코드 품질과 재사용성

마지막으로는 **컴포넌트 구조 개선**에도 신경 썼습니다. 뉴스 카드, 대시보드, 리포트 페이지에서 중복되던 기사 리스트 UI를 `ArticleRow` 컴포넌트로 통합하고, 신문 섹션 헤더 역시 `SectionHeader`로 추출했습니다. React.memo로 최적화해서 30개 카드가 한 번에 렌더링될 때의 성능도 개선했고요.

```tsx
const ArticleRow = memo(({ article, showDraft = true }) => {
  const isHigh = isHighImportance(article);
  return (
    <Card className={isHigh ? "border-l-4 border-l-primary" : ""}>
      {/* ... */}
    </Card>
  );
});
```

## 결과와 배운 점

단순한 "리스타일링"으로 시작했지만, 결과적으로는 **사용자 경험 전반의 재설계**가 되었습니다. 브랜드 정체성을 살리면서도 실제 뉴스룸 업무에 최적화된 인터페이스가 탄생했죠. 특히 신문 편집 관례를 디지털로 옮겨오는 과정에서, 전통 미디어의 노하우가 여전히 유효하다는 점을 새삼 느꼈습니다.

가장 중요한 교훈은 **"사용자가 누구인지 명확히 하라"**였습니다. Generic한 대시보드는 누구에게나 익숙할 수 있지만, 특정 도메인의 전문가들에게는 오히려 불편할 수 있거든요. 서울신문 기자들에게 최적화된 UI를 만드는 과정이, 결국 더 나은 제품을 만드는 길이었습니다.
