## 왜 'MBTI'를 지웠나

처음 서비스를 설계할 때 핵심 컨셉은 명확했다. 사주(타고난 것)와 MBTI(후천적으로 형성된 것)를 결합해 사용자를 더 입체적으로 해석한다는 것. 그런데 한동안 운영하면서 이상한 점을 발견했다. 화면 곳곳에 `사주 × MBTI`, `후천 기질`, `MBTI 코드`가 박혀 있는데, 정작 사용자가 경험하는 건 열 가지 상황극 선택지였다. 내부 구현 언어가 그대로 UI에 흘러나와 있었다.

이번 커밋의 출발점은 거기였다. 기술 스택 설명이 아닌, 사용자가 실제로 경험하는 여정을 언어로 담자.

## 브랜드 언어 전환: MBTI → 살아오며 익힌 성향

변경의 핵심은 두 가지다.

**'MBTI'라는 외부 검사 체계 참조를 제거**하고, **'결'이라는 추상 메타포를 구체적인 언어로 교체**했다.

```tsx
// Before
<p className="t-label font-bold text-cheong-deep">지금의 나 · 후천 기질</p>

// After  
<p className="t-label font-bold text-cheong-deep">살아오며 익힌 나의 성향</p>
```

`사주 × MBTI`는 `태어난 기운 × 지금의 성향`이 됐다. `후천 기질`은 `살아오며 익힌 성향`이 됐다. `MBTI 코드 직접 입력` 기능은 아예 제거했다. 이 기능이 있다는 것 자체가 "우리는 MBTI를 입력받는 서비스"라는 인상을 줬기 때문이다.

```tsx
// 삭제된 코드 — MBTI 직접 입력 UI 전체 제거
<button type="button" onClick={() => setManual((m) => !m)}>
  내 기질 유형을 알고 있어요 (직접 입력)
</button>
{manual && (
  <input placeholder="ENFP" maxLength={4} onChange={handleManualChange} />
)}
```

대신 퀴즈를 마친 후 나오는 메시지도 바꿨다. `간단 유사 진단으로, 공식 MBTI 검사가 아닙니다`라는 면책 문구는 사라지고, `한 번의 선택보다, 반복해서 마음이 향한 쪽을 읽었어요`라는 서비스 고유의 설명이 들어갔다.

## '결'이라는 단어와의 싸움

흥미로운 건 두 번째 커밋이다. 첫 번째 전환에서 `MBTI`를 빼면서 `결`이라는 단어를 많이 썼는데, 이번에는 그 `결`마저 걷어냈다.

`결`은 원래 나무의 결처럼 고유한 속성을 뜻하는 세계관 언어였다. 그런데 쓰다 보니 성향도 결, 관계도 결, 감정도 결, 흐름도 결이 됐다. 만능 추상 명사로 변질된 것이다.

이를 막기 위해 아예 **Quality Gate에 '결' 감지 로직**을 추가했다.

```typescript
// lib/llm/brand-voice.ts
const ABSTRACT_GYEOL_PATTERN = 
  /(?:^|[\s"'""''([{])결(?:이|을|은|의|로|에|도)?(?=$|[\s.,!?…"'""'')\]}])/;

/** 일반어 결정·결과·결혼은 허용하고, 만능 메타포로 홀로 쓰인 '결'만 찾는다. */
export function containsAbstractGyeol(text: string): boolean {
  return ABSTRACT_GYEOL_PATTERN.test(text);
}
```

정규식이 정교하다. `결정`, `결과`, `결혼`처럼 합성어로 쓰인 경우는 허용하고, 공백이나 조사 앞뒤에 홀로 서있는 `결`만 잡아낸다. 이 함수는 Basic과 Premium 두 Quality Gate 모두에 연결되어 LLM이 생성한 텍스트에서 추상적 `결`이 발견되면 재생성을 트리거한다.

브랜드 가이드(`BRAND_VOICE_GUIDE`)에도 명시적으로 추가했다. `바탕·기운·성향·방식·리듬·습관처럼 문맥에 맞는 구체적인 말을 고른다.`

## 연령별 프롬프트 보완: 나이 단정 금지

같은 커밋에서 프롬프트 구조도 크게 바뀌었다. 기존에는 나이대(`ageToneSeed`)를 `themeHint`, `toneHint`, `adviceFocus` 세 가지로만 LLM에 전달했는데, 이번에 `primaryFocus`, `secondaryFocus`, `sceneHints`, `avoidAssumptions`가 추가됐다.

핵심은 `avoidAssumptions`다. 30대라도 미혼일 수 있고, 20대라도 취업 중이 아닐 수 있다. 나이만으로 삶의 상태를 단정하는 것은 콜드 리딩과 다름없다. 이제 LLM 프롬프트에는 이렇게 전달된다.

```
단정 금지: 직업/취업 여부 / 혼인 여부 / 자녀 유무
```

섹션 설명도 구체화됐다. `사랑의 기본 리듬`이 아니라 `친밀감·애정·돌봄의 기본 리듬`, `일과 돈을 다루는 방식`이 아니라 `배움·일·기여·돈을 다루는 방식`. 삶의 각 영역이 특정 생애 단계를 전제하지 않도록 범위를 넓혔다.

## 친구 연결 기능 임시 비활성화

한 가지 더. 친구 리포트를 연결해 궁합을 보는 기능을 이번에 잠시 껐다.

```typescript
// 친구 리포트 결합은 연결 동의·철회 정책이 마련될 때까지 공개하지 않는다.
const FRIEND_COMPAT_ENABLED = false;
```

기능 자체는 완성되어 있지만, 상대방의 동의와 연결 철회 정책이 정비되기 전에 공개하는 건 옳지 않다고 판단했다. 카드 공유는 살리고 친구 연결만 껐다. 플래그 하나로 깔끔하게 처리했고, 코드는 그대로 남아있다.

---

이번 작업의 본질은 기능 개발이 아니라 언어 정제였다. 내부에서 쓰는 기술 용어가 사용자 화면에 노출되고 있지 않은지, 추상 표현이 너무 많이 쓰이고 있지 않은지 점검하는 것. 그리고 그 기준을 Quality Gate로 자동화한 것. 브랜드 일관성은 디자인 시스템이나 컴포넌트가 아니라 이런 반복적인 언어 교정에서 만들어진다.
